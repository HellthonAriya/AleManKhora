## ⚡ نصب با یک دستور

روی سرور (Ubuntu/Debian/CentOS/Fedora/Arch/Alpine) کافی است این دستور را اجرا کنی:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/HellthonAriya/AleManKhora/claude/wrongway-web-game-mfjws8/install.sh)
```

اسکریپت نصب به‌صورت خودکار:

- پیش‌نیازها (`git`، `curl`) و **Node.js ≥ ۱۸** را نصب می‌کند (در صورت نبود، با NodeSource یا nvm).
- پروژه را کلون می‌کند و وابستگی‌ها را نصب می‌کند.
- یک فایل `.env` با کلید امنیتی تصادفی می‌سازد.
- اگر روت باشی و systemd موجود باشد، یک **سرویس systemd** می‌سازد و اجرا می‌کند؛ وگرنه با اسکریپت مدیریت بالا می‌آورد.

> می‌توانی با متغیرها سفارشی کنی، مثلاً پورت و مسیر:
> ```bash
> AMK_PORT=8080 AMK_DIR=/opt/amk bash <(curl -fsSL .../install.sh)
> ```

## 🛠 اجرای دستی (توسعه)

```bash
git clone https://github.com/HellthonAriya/AleManKhora.git
cd AleManKhora
npm install
cp .env.example .env      # و JWT_SECRET را عوض کن
npm run seed              # (اختیاری) ادمین و چند بازیکن نمونه
npm start                 # → http://localhost:3000
```

- اولین کاربری که ثبت‌نام کند، خودکار **مدیر** می‌شود.
- بعد از `npm run seed` حساب آماده است: **`admin` / `admin123`**.

## 🎛 کنسول مدیریت — `manage.sh`

یک منوی کامل برای ادارهٔ سرور؛ بدون آرگومان منوی تعاملی باز می‌شود:

```bash
cd AleManKhora
./manage.sh
```

| دستور | کار |
|------|-----|
| `./manage.sh install`   | نصب وابستگی‌ها و ساخت `.env` |
| `./manage.sh update`    | `git pull` + نصب مجدد + ری‌استارت |
| `./manage.sh start`     | اجرای سرور (systemd یا nohup) |
| `./manage.sh stop`      | توقف |
| `./manage.sh restart`   | ری‌استارت |
| `./manage.sh status`    | وضعیت اجرا |
| `./manage.sh logs`      | مشاهدهٔ زندهٔ لاگ‌ها |
| `./manage.sh backup`    | **بکاپ کامل** (دیتابیس + `.env` + متادیتا) در `backups/` |
| `./manage.sh restore`   | **بازیابی** از یک بکاپ (با انتخاب از فهرست) |
| `./manage.sh port 8080` | تغییر پورت |
| `./manage.sh seed`      | ساخت دادهٔ نمونه/ادمین |
| `./manage.sh uninstall` | حذف سرویس و توقف برنامه |

> بکاپ یک فایل `amk-backup-<تاریخ>.tar.gz` می‌سازد که کل وضعیت سایت (پایگاه‌داده و تنظیمات) را در خود دارد؛ کد روی گیت است.

## ✨ امکانات بازی

### 🎮 حالت‌های بازی
- 🎲 **بازی تصادفی (Matchmaking)** — همگام‌سازی آنی با حریف هم‌سطح (۲ یا ۴ نفره).
- 🤝 **دعوت دوست** — اتاق خصوصی با کد ۶ حرفی و لینک دعوت.
- 🤖 **بازی با هوش مصنوعی** — سه سطح سختی؛ در حالت ۴ نفره با چند ربات.
- 👤 **مهمان یا حساب کاربری** — بدون ثبت‌نام بازی کن یا با حساب امتیاز جمع کن.

### 👥 بازی ۲ و ۴ نفره
- حالت کلاسیک **۲ نفره** (روبه‌رو) و حالت **۴ نفره** (هر بازیکن از یک ضلع).
- در ۴ نفره هر بازیکن باید به ضلع مقابل برسد؛ تسلیم/قطع‌شدن/اتمام‌زمان باعث **حذف** بازیکن می‌شود و بازی ادامه پیدا می‌کند تا یک نفر بماند یا کسی به هدف برسد.

### ⏱ تایمر شطرنجی
- کنترل زمان قابل‌انتخاب: بدون زمان، ۱، ۳، ۵ یا ۱۰ دقیقه، با **پاداش زمانی** اختیاری هر حرکت.
- ساعت‌ها روی سرور معتبرند و در رابط کاربری زنده شمارش معکوس می‌شوند؛ اتمام زمان = حذف/باخت.

### 📺 تماشای زنده
- در سالن، فهرست **بازی‌های زندهٔ در حال پخش** نمایش داده می‌شود.
- با یک کلیک به‌صورت **تماشاگر** وارد بازی می‌شوی و حرکت‌ها را بلادرنگ می‌بینی.

### 🏆 رقابت و پروفایل
- **جدول رتبه‌بندی ELO**، صفحهٔ پروفایل با آمار و تاریخچهٔ بازی‌ها (برای ۲ و ۴ نفره).
- گفتگوی درون‌بازی، اتصال مجدد پس از قطعی، و بازی مجدد (Rematch).

### 🎨 گرافیک و سفارشی‌سازی
- صفحهٔ **HTML5 Canvas** با انیمیشن حرکت مهره، افکت ساخت دیوار، هایلایت حرکت‌های مجاز و پیش‌نمایش زندهٔ جای دیوار.
- سفارشی‌سازی هر بازی: تعداد بازیکن (۲/۴)، اندازهٔ صفحه (۵×۵ تا ۱۱×۱۱)، تعداد دیوارها، **تایمر**، **۶ تم رنگی** و **رنگ هر بازیکن**.

### 🛡 پنل مدیریت کامل
- داشبورد آمار زنده، مدیریت کاربران (مسدودسازی/مدیر/حذف/تغییر رمز)، فهرست بازی‌ها و پیکربندی سراسری سایت.

## 🧩 پشتهٔ فنی

| بخش | فناوری |
|-----|--------|
| سرور | Node.js + Express |
| بلادرنگ | Socket.IO |
| پایگاه‌داده | SQLite (`better-sqlite3`) |
| احراز هویت | JWT (کوکی httpOnly) + bcrypt |
| کلاینت | Vanilla JS (ES Modules) + HTML5 Canvas — **بدون مرحلهٔ build** |
| موتور بازی | یک نسخهٔ واحد در `public/js/engine.js`، مشترک بین سرور و مرورگر |

## 📜 قوانین بازی

- هر بازیکن یک مهره دارد که از یک ضلع شروع می‌شود و باید به **ضلع مقابل** برسد.
- در هر نوبت یکی از این دو کار را انجام بده:
  1. **حرکت** مهره به خانهٔ مجاور (با امکان پرش از روی حریف، و پرش مورب وقتی پشت حریف بسته است).
  2. **ساخت دیوار** بین خانه‌ها برای کند کردن حریف.
- دیوار را نمی‌توان طوری گذاشت که مسیر **هیچ** بازیکنی به‌طور کامل بسته شود (با جست‌وجوی مسیر بررسی می‌شود).
- هر بازیکن تعداد محدودی دیوار دارد.
